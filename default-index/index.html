new/delete/malloc/free

发表于 2022-03-30 分类于 C++

为什么要引入new和delete

C++语言定义了两个运算符来分配和释放动态内存（一般是堆内存）。

运算符new分配内存，delete释放new分配的内存。

为什么不用malloc和free

由于malloc/free是库函数而不是运算符，不在编译器控制权限之内，不能够把类的执行构造函数和析构函数的任务强加于malloc/free。因此C++语言需要一个能完成动态内存分配和初始化工作的运算符new，以及一个能完成清理与释放内存工作的运算符delete。

拷贝构造函数的参数为什么是const&

发表于 2022-03-29 分类于 C++

拷贝构造函数的参数为什么是const&
为什么有&:

在传递对象之前，编译器需要创建该对象的副本。

因此，编译器为了处理拷贝构造函数的这条调用语句，需要调用拷贝构造函数来创建实参的副本。但是，由于非&是按值传递，第二次调用同样需要创建实参的副本，因此需要再一次调用拷贝构造函数，最终得到的是对拷贝构造函数的无限调用。

于是，拷贝构造函数必须要带引用类型的参数，而且这也是编译器强制性要求的。

为什么有const:

如果函数不会改变引用类型参数的值，加不加const的效果是一样的。不加const，编译器也不会报错。但是为了整个程序的安全，还是加上const，防止对引用类型参数值的意外修改。

深拷贝与浅拷贝

发表于 2022-03-29 更新于 2022-03-30 分类于 C++

前提
在类对象进行拷贝的过程中，如果类中没有自定义拷贝构造函数，那么系统会提供一个缺省的拷贝构造函数。缺省的拷贝构造函数对于基本类型的成员变量，按字节复制；对于类类型成员变量，调用其对应的拷贝构造函数。

浅拷贝
缺省的拷贝构造函数就是浅拷贝，它只拷贝对象，不拷贝对象所指的资源。
如果类成员中含有指针变量，那么缺省的拷贝构造函数和赋值函数就隐含了错误：

  1. 被拷贝的（原有的/左边的）内存空间没有被释放，造成内存泄漏；
  2. 被拷贝对象与拷贝对象中的两个指针均指向同一块内存空间，用一个指针去改变所指内存空间的内容时，另一个指针所指内存空间内容也会被改变；
  3. 当被拷贝对象与拷贝对象中的两个指针被析构时，一共调用了两次析构函数去释放同一块内存空间。第一个指针被析构时，它们所指的内存空间已经被释放。第二次调用析构函数时，释放一个已经被释放的内存空间，将发生未定义的结果。（系统崩溃、内存泄漏等等）

此时就需要深拷贝。

深拷贝
拷贝类对象时，不仅仅拷贝类对象本身，还拷贝类对象所指的资源。
要想实现深拷贝，必须显式地定义拷贝构造函数和赋值运算符。
如果想把对象A拷贝到对象B中时，显式定义拷贝构造函数/赋值运算符时，可以先创建一个临时对象temp，此时编译器会调用构造函数开辟一段和A一样大的空间，然后把A的内容放在temp所指空间中，再交换B和temp对象的内容，最后用析构函数销毁临时对象temp，就完成了将对象A拷贝到对象B中去的工作。

什么时候会调用拷贝函数呢？

  1. 对象以值传递（非引用传递）的方式传入函数体；
  2. 对象以值传递（非引用传递）的方式从函数返回；
  3. 一个对象需要通过另一个对象初始化。
